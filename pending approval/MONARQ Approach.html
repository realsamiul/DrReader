A Superior Engineering Approach
A generic model cannot solve a specific, nuanced problem. M0NARQ’s architecture abandons the monolithic model for an intelligent, dual-bucket system. We use the "right model for the right job"—a strategy that surgically minimizes cost while maximizing accuracy and compliance.
Bucket 1: The "Eyes"

Bucket 1: The "Eyes" – Specialist Vision Models (VLMs)

This is the "Eyes". Instead of one weak, generic model, we route tasks to a stack of global, SOTA specialists. We use Anthropic's Claude 3.5 Sonnet for the handwritten cursive that Google deems "Experimental", NVIDIA's Nemotron for high-fidelity tables/MICR codes, and IBM's Docling for 30x faster multi-page PDF parsing.
Specialist for Handwriting (92% Accuracy)
Specialist for Tables/MICR (97% Accuracy)
Specialist for Multi-Page Speed (30x Faster)
Specialist for Printed Bengali (97% Accuracy)
Bucket 2: The "Brain"

Bucket 2: The "Brain" – Native Bengali LLMs

This is our most significant differentiator. After the "Eyes" read the text, our "Brain" *understands* it. This is the Bengali reasoning layer that AWS and Google lack. It validates, cross-references, and detects fraud.
Amount Parsing: Parses "পঞ্চাশ হাজার টাকা" → 50,000 using TituLLM-3B.
Name Validation: Matches "মোঃ করিম" → "Md. Karim" using BanglaBERT from BUET CSE.
Fraud Reasoning: Detects anomalies across documents using Qwen3-235B.
Translation: Handles Bengali-English translation using Meta's Llama 3.1-8B.
The Result: A System That Thinks, at 3.3x Lower Cost.

This dual-bucket system is not just clever design; it is a direct exploit of our competitors' documented gaps. It solves the handwriting problem, adds the missing reasoning layer, and leverages open-source models to break the 3.3x cost disparity.
